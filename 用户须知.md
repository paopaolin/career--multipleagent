# 📘 用户须知

> 职业规划多智能体协同系统 — 新用户上手指南

---

## 1. 这是什么？

一个基于 Claude Code 构建的 **AI 职业规划助手**。它不像普通聊天机器人那样一问一答，而是同时派出多个 AI 专家为你工作——一个分析你的技能差距，一个帮你搜职位，一个给你制定学习路线，一个帮你准备面试。

你说一句话，它给你一份完整报告。

---

## 2. 它能做什么？

| 场景 | 你只需要说 | 它会做什么 |
|------|-----------|-----------|
| 🎯 **完整规划** | "我是XX，想转YY，帮我规划" | 技能分析 + 职位搜索 + 学习路线 + 面试准备，一份综合报告 |
| 🔍 **技能差距** | "分析我和XX岗位的差距" | 对比你的技能 vs JD 要求，给出匹配度评分和优先级建议 |
| 💼 **找职位** | "帮我搜北京30k以上的Go岗位" | 从 5 个数据源并行搜索，按匹配度排序，附带薪资趋势 |
| 🗺️ **学习路线** | "帮我制定AI工程师学习计划" | 周级路线图 + 课程推荐 + 实战项目序列 |
| 🎤 **面试准备** | "帮我准备腾讯后端的面试" | 技术题 + 行为题 + 系统设计题 + STAR 框架 + 反问清单 |

---

## 3. 团队里的 AI 们

| 角色 | 负责什么 |
|------|---------|
| 🎯 **监督者** | 听懂你的需求，分配任务给专家，最后把结果汇总成报告 |
| 🔍 **角色分析师** | 盘点你的技能，对比岗位要求，找出差距 |
| 💼 **求职搜索师** | 同时搜索 5 个平台（JobDataLake、HireJack、JD Intel、LinkedIn、公司数据库） |
| 🗺️ **学习规划师** | 根据差距定制学习路线，推荐课程和项目 |
| 🎤 **面试教练** | 生成面试题库，教你用 STAR 方法回答问题 |

> 你不需要手动选择找谁——监督者会自动判断。

---

## 4. 你会得到什么样的结果？

每次完整规划结束后，你将收到一份结构化的报告，包含：

```
📋 职业规划报告
 ├── 📊 现状评估    → 你的技能画像 + 优势 + 不足
 ├── 🎯 市场分析    → 匹配职位列表 + 薪资范围 + 热门技能
 ├── 🗺️ 学习路线    → 分阶段的周计划 + 推荐课程/书籍/项目
 ├── 🎤 面试准备    → 针对性的面试题 + 回答框架 + 模拟脚本
 └── 📋 行动建议    → 优先级排序的下一步行动清单
```

## 5. 职位数据从哪来？

系统接入了 5 个数据源，覆盖国内外主流招聘渠道：

| 数据源 | 覆盖 |
|--------|------|
| JobDataLake | 100 万+ 职位，2 万+ 公司，每小时更新 |
| HireJack | 500+ 科技公司，8 万+ 实时职位 |
| JD Intel | Greenhouse、Lever、Ashby 等 7 个 ATS 平台直招 |
| LinkedIn Jobs | LinkedIn 全球职位库 |
| Company Research | 公司背景、融资、文化调查 |

> 同时也支持搜索国内平台（Boss直聘、猎聘、拉勾等）。

---

## 6. 它会记住你

系统会通过知识图谱自动记住：

- 你的技能、经验、学历
- 你的职业目标和偏好
- 过往的分析结果
- 学习进度

**好处**：下次使用不用重复介绍自己，它会基于历史记录给出更精准的建议。

**隐私**：所有数据存储在本地 `memory/` 目录，不会上传到云端。

## 7. 环境配置（重要）

> ⚠️ 本项目依赖多个以 `.` 开头的配置文件（隐藏文件），GitHub 可能不会上传这些文件。克隆项目后，请先运行初始化脚本。

### 一键配置（推荐）

```bash
# 克隆项目后，运行初始化脚本
bash setup.sh
```

该脚本会自动创建以下所有配置文件：

```

├── .mcp.json                 # MCP 数据服务器配置（5个数据源）
├── .gitignore                # Git 忽略规则
├── .claude/
│   ├── settings.json         # 项目权限与记忆路径配置
│   ├── agents/               # 5 个智能体定义
│   │   ├── supervisor.md
│   │   ├── role-analyzer.md
│   │   ├── learning-path-creator.md
│   │   ├── job-search-agent.md
│   │   └── interview-preparer.md
│   └── skills/
│       └── career-planning.md
└── .company-research-mcp/    # 公司调查 MCP 项目
```

## 8. 怎么使用？

### 第一步：打开 Claude Code

```bash
cd 项目目录
claude
```

### 第二步：描述你的需求

越具体越好，比如：

```
✅ 好的描述：
"我是3年React前端开发，熟悉TypeScript和Node.js，
想6个月内转AI工程师，base北京，期望薪资35k+"

❌ 不够的描述：
"帮我看看工作"
```

### 第三步：回答监督者的追问

监督者可能会问你 1-2 个补充问题（比如时间规划、地点偏好），如实回答即可。

### 第四步：等待报告生成

系统会自动调度专家，通常 1-3 分钟完成完整规划。

## 9. 常见问题

**Q: 需要付费吗？**
A: 完全免费。所有 MCP 数据源都有免费额度（每天 500 次搜索足够个人使用）。

**Q: 需要什么前置条件？**
A: 安装 [Claude Code](https://docs.anthropic.com/zh-CN/docs/claude-code/overview) CLI 和 Node.js >= 18。

**Q: 搜索结果是实时的吗？**
A: 是的。JobDataLake 每小时更新，HireJack 和 JD Intel 实时抓取。

**Q: 支持哪些城市？**
A: 全球主要城市都支持，包括北上广深杭等国内城市，以及远程岗位。

**Q: 我的数据安全吗？**
A: 所有用户数据（技能、简历信息、分析结果）仅保存在本地，不会上传到任何第三方服务器。记忆系统使用本地知识图谱存储。

**Q: 可以同时让多个专家工作吗？**
A: 这正是本系统的核心设计。监督者会自动判断哪些任务可以并行，最大化效率。

**Q: 如果我只想问一个问题呢？**
A: 没问题。监督者会识别简单查询，直接路由到对应专家，不会启动完整流程。

## 10. 技巧与小贴士

1. **第一次使用建议做完整规划**——这样系统能全面了解你，后续使用会更精准
2. **善用记忆功能**——告诉系统你的技能和偏好，它会越用越懂你
3. **随时纠正**——如果专家输出有偏差，直接告诉监督者，它会重新调度
4. **组合使用**——可以先做完整规划，拿到学习路线后，隔几周再搜一次职位看市场变化
