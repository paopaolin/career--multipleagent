# 职业规划多智能体协同系统

## 项目概述

本项目构建一个多智能体协同的职业规划系统。由**监督者智能体**作为中央协调器，负责任务理解、路由和结果汇总；多个**专家智能体**各司其职，并行或顺序完成专业任务。

## 智能体团队

| 智能体 | 角色 | 模型 | 触发场景 |
|--------|------|------|----------|
| 🎯 `supervisor` | 中央协调器 | Fable | 所有对话入口，理解意图并路由 |
| 🔍 `role-analyzer` | 角色分析专家 | Sonnet | 分析技能差距、评估岗位匹配度 |
| 🗺️ `learning-path-creator` | 学习路径规划专家 | Sonnet | 生成定制化学习 Roadmap |
| 💼 `job-search-agent` | 求职搜索专家 | Sonnet | 联网搜索匹配职位 |
| 🎤 `interview-preparer` | 面试准备专家 | Sonnet | 生成面试问题和准备材料 |

## 协同工作流

### 标准流程（并行分工模式）

```
用户描述职业目标
       │
       ▼
🎯 监督者：理解意图 → 收集必要信息 → 制定执行计划
       │
       ├─→ 🔍 角色分析专家：分析现状 vs 目标岗位差距
       ├─→ 💼 求职搜索专家：搜索当前市场匹配职位   │ 并行
       │
       ▼（基于分析结果）
       ├─→ 🗺️ 学习路径规划专家：生成学习路线图
       ├─→ 🎤 面试准备专家：生成面试准备材料       │ 并行
       │
       ▼
🎯 监督者：汇总所有结果 → 生成综合职业规划报告
```

### 简化流程

```
用户问题 → 🎯 监督者判断 → 路由到单个专家 → 返回结果
```

## 记忆与状态管理

### 持久化记忆（跨会话）
使用 Memory MCP Server (知识图谱) 存储：
- **用户画像**：技能、经验、学历、兴趣
- **职业目标**：目标岗位、期望薪资、地点偏好
- **历史记录**：过往分析结果、已申请的职位
- **学习进度**：已完成的学习里程碑

### 会话状态（当前对话）
- 当前任务上下文
- 各智能体的中间结果
- 用户反馈和修正

## MCP 数据工具链

项目配置了 3 个专用 MCP 服务器（`.mcp.json`）用于职位数据检索：

| MCP 服务器 | 核心能力 | 免费额度 | 需要认证 |
|-----------|---------|----------|----------|
| **JobDataLake** | 1M+ 职位、20,000+ 公司、AI 标注薪资、技能标签 | 500 次/天 | 否 |
| **HireJack** | 科技岗位搜索、市场脉动、公司技术栈分析 | 公开工具免费 | 否（Pro需认证） |
| **JD Intel** | 跨 ATS 搜索（Greenhouse/Lever/Ashby/Workday） | 免费 | 否 |

> 💡 MCP 工具在 Claude Code 重启后自动加载。工具名格式：`mcp__<server>__<tool>`

## 使用示例

```
用户: 我是前端开发，想转AI工程师，帮我规划一下
→ 监督者启动完整流程

用户: 帮我分析一下我的技能和字节跳动AI产品经理岗位的差距
→ 监督者路由到角色分析专家

用户: 帮我搜索北京地区的高级Java开发岗位
→ 监督者路由到求职搜索专家
```
